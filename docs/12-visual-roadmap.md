# 12 - Visual Roadmap to Expected Quality

## Goal

Reach a visual standard defined by:

- painterly warmth
- elegant silhouettes
- strong atmosphere
- HUD restraint
- expressive cats
- layered stations with depth
- smooth, meaningful motion

This roadmap is not about copying any reference literally.
It is about reaching a similar level of finish, softness, readability, and emotional tone.

## Visual Quality Gap Today

Current weaknesses:

- placeholder geometry dominates the playfield
- train, cats, props, and enemies still read as primitives
- HUD is still heavier than the art
- scene depth is limited
- atmosphere depends more on flat color than layered painting
- typography and framing still read as prototype UI

## Execution Rules

- do not redesign all screens at once
- use the prologue as the visual benchmark
- only move to the next sprint after one screen looks clearly better in screenshot form
- keep desktop and mobile review in every sprint
- every sprint must end with new screenshots for comparison

## Sprint V1 - Visual Direction Lock

Goal:

- freeze the language before more implementation work

Tasks:

- define final palette families for day, dusk, night, and memory scenes
- define typography pair for title and interface
- define materials:
  - wood
  - aged metal
  - stitched cloth
  - warm lantern glass
  - cold station stone and steel
- define shape rules for:
  - cats
  - train
  - dogs
  - humans
  - props
- produce 3 target mockups:
  - prologue
  - train hub
  - station run

Deliverables:

- final moodboard
- color system
- UI reference sheet
- composition references

Definition of done:

- no unresolved disagreement about visual direction
- one screenshot target exists for each primary scene

Estimated solo effort:

- 3 to 5 days

## Sprint V2 - HUD Restraint and Layout Pass

## What Must Not Happen

- no new large systems mixed into visual sprints
- no extra routes before route 1 looks good
- no UI densification during art polish
- no "temporary" art pass that becomes permanent by inertia

## Recommended Working Rhythm

Per sprint:

1. capture current screenshots
2. choose one visual problem only
3. implement pass
4. compare before/after
5. test desktop and mobile
6. document what became the new standard

## Success Condition

The project reaches the expected result when:

- the prologue looks emotionally strong on first boot
- the hub feels warm and memorable
- each run screenshot has atmosphere and depth
- cats feel expressive and readable
- UI no longer dominates the image
- mobile still preserves the mood
